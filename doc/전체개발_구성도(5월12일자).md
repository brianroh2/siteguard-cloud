## 2. 데이터 흐름

### 2-1. 라이브 영상 흐름

```
[카메라 RTSP] ──(LTE DDNS)──► [go2rtc, Hetzner] ──(WebSocket/MSE)──► [관리자 브라우저]
  CCTV-1: 0004312.m2mnet.kr:554/profile1
  CCTV-3: 0004312.m2mnet.kr:555/Ch1
  전송방식: exec:ffmpeg -rtsp_transport tcp (TCP 강제)
```

### 2-2. 감지 이벤트 흐름

```
[카메라] ──(RTSP pull)──► [Frigate] ──(감지 이벤트)──► [Mosquitto:1883]
                                                              │
                                                    [frigate_tb_bridge.py]
                                                              │ MQTT
                                                              ▼
                                                    [ThingsBoard:1884]
                                                    텔레메트리 키:
                                                    - online, cameras_online
                                                    - cam_cctv-1_fps, cam_cctv-1_online
                                                    - detect_events_today
                                                    - inference_speed
```

### 2-3. ONVIF 펌웨어 조회 흐름

```
[관리자] 🔄 ONVIF 갱신 클릭
    ▼
[TB 클라우드] RPC: getOnvifInfo 전송
    ▼ MQTT
[edge_onvif_handler.py]
    ▼ HTTP Digest (포트 8899)
[카메라 ONVIF]
    ▼
[TB SERVER_SCOPE 속성 갱신]
firmware_version, firmware_date
```

### 2-4. 비상벨·음성 통화 흐름 (예정)

```
[현장 비상벨 버튼] ──► [RPi Baresip SIP 클라이언트]
                               │ SIP (인터넷)
                               ▼
                    [Asterisk 서버, Hetzner]
                               │
                               ▼
                    [관리자 단말 SIP 수신]
                    또는 [현장 스피커 방송]
```
