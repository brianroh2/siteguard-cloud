# SIP 음성통화 기능 개발 계획

> 작성일: 2026-04-23
> 상태: 기획/설계 단계 (CCTV 관제 프로젝트와 별도 POC 진행 후 통합 예정)

---

## 1. 개요

이동형 CCTV 단말(LTE 라우터 기반)에 **비상벨 호출 + 양방향 음성통화** 기능을 추가한다.
영상(CCTV)은 기존 go2rtc/Frigate 파이프라인을 그대로 사용하고,
음성통화는 SIP 프로토콜 기반 자체 PBX(Asterisk)를 클라우드에 구축하여 제공한다.

### 핵심 설계 원칙

- **동적 IP 대응**: LTE 고정IP 없이도 동작 — Tailscale로 에지→클라우드 outbound 연결
- **별도 번호 불필요**: 자체 Asterisk 내선만으로 통화 (필요 시 070 트렁크 추가)
- **경량 에지**: RPi 2도 동작 가능한 수준으로 설계 (영상은 IP카메라가 담당)
- **단계적 통합**: POC 완료 후 현재 관제 시스템에 통합

## 3. 호출 흐름

### 3-1. 비상벨 호출 (에지 → 관리자)

```
1. 현장 사람이 비상버튼 누름
2. RPi GPIO 감지 → Python 스크립트 실행
3. Baresip으로 Asterisk 내선 호출 (예: 101번)
4. Asterisk → 관리자 Linphone 앱 벨 울림
5. 관리자 수신 → 마이크/스피커 양방향 통화
6. 동시에 관제 UI에서 현장 카메라 영상 확인
```

### 3-2. 관제 → 현장 방송 (관리자 → 에지)

```
1. 관리자가 관제 UI에서 방송 버튼 클릭
2. Asterisk가 에지 RPi(내선 201번)로 자동 발신
3. RPi Baresip 자동 수신 설정
4. 관리자 음성 → 에지 스피커 출력 (안내방송/경고)
```

### 3-3. 알람 사이렌

```
1. Frigate AI 감지 이벤트 발생
2. TB-2 브리지 → Thingsboard 텔레메트리 전송
3. 관제 UI 알람 발생 → Asterisk로 트리거 (추후)
4. RPi aplay → 사이렌 사운드 재생
```

## 4. 필요 하드웨어

### 4-1. 에지 단말 (현장)

| 부품 | 모델 (예시) | 가격 (참고) | 비고 |

| WiFi 동글 | USB WiFi (802.11n) | ~5,000원 | RPi 2 WiFi 없음 |
| USB 사운드카드 | TROND 또는 동급 | ~3,000원 | 마이크+스피커 동시 |
| 마이크 | 단일지향성 USB/3.5mm | ~5,000원~ | 노이즈 환경 고려 |
| 스피커/사이렌 | USB 액티브 스피커 | ~10,000원~ | 실외 설치 시 방수 고려 |

| 케이스 | 방수 인클로저 | ~15,000원~ | 옵션 |

## 6. 단계별 추진 계획

### Phase 1: 클라우드 SIP 서버 구축 (Asterisk)

**목표:** Hetzner에 Asterisk 설치, 내선 통화 테스트

```
작업:
- docker-compose에 Asterisk 컨테이너 추가
- 내선 설정: 101 (관리자), 201 (에지 RPi)
- 방화벽: UDP 5060, RTP 10000-20000 개방
- 테스트: PC SIP 클라이언트 ↔ 스마트폰 Linphone 통화

완료 조건: PC와 스마트폰 간 SIP 음성통화 성공
```

### Phase 2: RPi 2 SIP 클라이언트 구성

**목표:** RPi에서 SIP 통화 송수신

```
작업:
- RPi OS 설치, Tailscale 연결
- Baresip 설치 및 Asterisk 내선 201 등록
- USB 사운드카드 + 마이크 + 스피커 연결 및 테스트
- RPi ↔ 스마트폰 Linphone 음성통화 확인

완료 조건: RPi 마이크/스피커로 스마트폰과 통화 성공
```

### Phase 3: 비상버튼 → 자동발신

**목표:** 버튼 누르면 관리자 폰으로 자동 연결

```
작업:
- GPIO 버튼 배선 및 Python 감지 스크립트
- 버튼 누름 → baresip 명령어로 101번 자동 발신
- systemd 서비스로 등록 (부팅 시 자동 시작)
- 통화 중 재호출 방지 로직

완료 조건: 버튼 → 관리자 폰 자동 연결, 3초 이내
```

### Phase 4: 사이렌 / 방송 수신

**목표:** 관제에서 명령 → 현장 스피커 출력

```
작업:
- Frigate 감지 이벤트 → 사이렌 재생 (aplay)
- Asterisk에서 RPi로 자동발신 → 방송 수신 설정
- 사이렌 음원 파일 준비 (wav)

완료 조건: AI 감지 시 사이렌 자동 재생
```

### Phase 5: 관제 UI 통합 (현재 프로젝트 연동)

**목표:** 기존 Next.js 관제 화면에 통화 기능 통합

```
작업:
- 관제 UI에 "방송" 버튼 추가 → Asterisk API 호출
- 기기 상세에 SIP 상태 표시 (등록 여부)
- 알람 발생 시 자동 호출 연동 (선택)
- 추후: 커스텀 모바일 앱으로 Linphone 대체

완료 조건: 관제 UI 버튼으로 현장 방송 가능
```

## 7. 현재 CCTV 관제 프로젝트 통합 방법

### 통합 포인트

```
현재 시스템                    SIP 기능 추가 후
──────────────────────────    ──────────────────────────────
기기 목록 (devices)        →  SIP 등록 상태 컬럼 추가
알람 페이지 (alarm)        →  알람 발생 시 자동 호출 연동
Device Groups             →  현장별 방송 버튼 추가
대시보드                   →  통화 중 현장 표시
```

### Asterisk REST API (ARI) 활용

Next.js에서 Asterisk를 직접 제어:

```typescript
// 관제 UI에서 현장 방송 예시
async function broadcastToSite(siteId: string) {
  await fetch(`http://localhost:8088/ari/channels`, {
    method: "POST",
    body: JSON.stringify({
      endpoint: `SIP/201`,  // 에지 RPi 내선
      callerId: "관제센터",
    })
  })
}
```

### Docker Compose 통합

```yaml
# 기존 docker-compose.yml에 추가
  asterisk:
    image: andrius/asterisk:latest
    ports:
      - "5060:5060/udp"   # SIP
      - "10000-10100:10000-10100/udp"  # RTP
    volumes:
      - ./asterisk/conf:/etc/asterisk
    restart: unless-stopped
```

## 10. 참고 / 보류 사항

- **커스텀 모바일 앱**: Phase 5 이후, 영상+통화+알람 통합 UI (React Native 또는 PWA)
- **SIP 영상통화**: Baresip + RPi 카메라 조합 가능하나 RPi 2 성능 한계로 Phase 5 이후 검토
- **다중 현장**: 내선 체계(2xx = 에지, 1xx = 관리자)로 확장 용이
- **PSTN 연결**: 070 SIP 트렁크 추가 시 일반 전화망으로 발신 가능
- **WebRTC 전환**: 추후 브라우저 기반 통화로 전환 시 Asterisk WebRTC 게이트웨이 활용 가능
